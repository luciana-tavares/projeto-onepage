Sobre mim

Olá! Meu nome é Luciana, também me chamam de Lu, Luc, Luzinha… Tenho 39 anos e sou de Recife/PE. Minha formação é Técnica em Administração, pela Escola Técnica Estadual Prof. Agamenom Magalhães (ETEPAM) e Bacharelado em Turismo, pela Universidade Federal de Pernambuco (UFPE). Algumas das minhas experiências profissionais foram: entreter e cuidar de crianças num shopping; atendimento aos turistas em postos de Informações Turísticas em Recife; participar de um projeto do Governo do Estado de PE, chamado Mãe Coruja, acompanhando a capacitação profissional de mães antes e após o nascimento dos seus filhos. Atualmente, sou empregada pública, trabalho na função de Assistente Administrativa numa coordenação de planejamento de transportes, especificamente do setor metroviário. Mesmo sendo “concursada”, como as pessoas costumam dizer, sempre fui uma pessoa muito curiosa, gosto bastante de ler, estudar conteúdos diversos, tais como fotografia, trabalhos manuais e, nesse último 1 ano e meio, despertei interesse pela área de tecnologia, passando a conhecer as possibilidades em Front-end, Back-end, já tenho noções de UX Design, Design Thinking, Metodologias Ágeis, etc… E foi exatamente esse despertar de interesse que me trouxe até o programa Elas na Tech. A história? Eu conto nos tópicos a seguir.

O começo

Em 2022, senti a necessidade de ultrapassar a barreira da atuação profissional administrativa. Eu queria conhecer outras áreas, afinal não pretendia me aprofundar na minha área de formação novamente. Comecei a pesquisar sobre tecnologia e me deparei com diversos cursos, vídeos no YouTube, então pensei: pode ser minha oportunidade! Fiz alguns, outros deixei pelo caminho, faltava algum estímulo. Corta para 2023. Passando pelo feed do Instagram, vi o anúncio do Elas na Tech, de imediato fui ler os critérios, estava tudo certo, vou me inscrever. E hoje, eu acredito que fazer a inscrição foi o primeiro desafio superado. Tem mais…

Desafios

Após a inscrição, veio a expectativa sobre ser aceita ou não para o programa. Tanta gente deve ter se inscrito, eu pensei. E pensei também, se eu for aceita, e agora? Vou conseguir acompanhar, entender? Será que vou desistir no caminho? Vai dar certo estudar EAD? E a professora, vai ajudar? (Obrigada, Gil e Erika! De coração…!) Superados os primeiros desafios, que são os mentais, com o início das aulas vieram os desafios de conteúdo. A cada semana um novo impacto (a maioria positivo, vale ressaltar), quando me deparava com a trilha que precisava ser seguida, o conjunto de aulas a serem assistidos, e ainda tinha os desafios que a professora solicitava de uma semana para a outra. Quem acompanhou de perto a minha trajetória ao longo desses meses sabe das noites, fins de semana e feriados na frente da tela para tentar desenvolver o que fosse possível. Sobre os meus desafios, eu posso dizer que todo dia ultrapassei um pouquinho, e foi assim que senti, pois seria mais fácil deixar tudo de lado e passar as horas livres na TV e no celular. Até mesmo agora, na reta final, tive férias do trabalho e tentei antecipar ao máximo a elaboração da página para não fazer às pressas... Devo confessar que, quando recebemos os critérios para montar a One Page, eu senti que talvez tivesse chegado o momento de abrir mão de tudo por não me achar capaz, mas em paralelo a este sentimento, repeti na minha mente muitas e muitas vezes: eu me recuso a desistir.

Conquistas

Ter conseguido fazer os desafios semanais propostos pela professora. Foi muito emocionante! Chegar até o final deste curso é a soma de conquistas diárias, semanais e mensais.

Aprendizados

Se eu for falar sobre o aprendizado teórico, começamos com as noções básicas de Internet e Programação, preparando o terreno para os próximos conteúdos. Em seguida, as primeiras aulas de Visual Studio Code foram um espetáculo. Eu já conhecia o editor de código, mas o curso abriu outra perspectiva de possibilidades, ferramentas, atalhos… Depois o Github, que apesar de assustar um pouco de cara, mostra-se uma plataforma tão poderosa para hospedagem de código e arquivos, possibilitando a colaboração de desenvolvedores do mundo inteiro. No meio do processo, chegamos até a trilha de desenvolvimento web Front-end: HTML, CSS e Javascript.

De tantas coisas que eu já estudei e fiz na vida, posso afirmar que essa trilha está entre as mais desafiadoras. Foi muito bom. Chegando na reta final, tivemos Bootstrap, mais um pouco de Lógica de Programação e Python. Tudo isso já pareceu mais leve, exceto pelo Bootstrap que ainda merece um pouco mais de dedicação… Python é agradável, pois já tinha feito alguns estudos sobre a linguagem no ano passado. Acredito que o conteúdo foi bem completo, eu só queria mais tempo, principalmente do programa Elas na Tech. O apoio da professora foi fundamental, pois a cada aula assistida, havendo qualquer dúvida, sempre pudemos contar com o suporte dela. De fato, tivemos uma dupla impecável (monitora e professora) na nossa turma 03 do curso.

Por fim, chegando ao momento de falar sobre o "aprendizado para a vida", o Elas na Tech foi uma soma de recomeços. Ao longo desses meses, toda semana precisávamos manter o fôlego, o ânimo, a coragem para continuar, porque no caminho surge um pouco de tudo para que você não vá adiante... Às vezes cansa, os erros sempre aparecem, você corrige, aprende e segue em frente. Eu pergunto: viver não é bem assim?!

Projetos

Projeto Layout HTML

Reprodução de layout HTML a partir do site Color Adobe Wheel. Recebemos um modelo para que pudéssemos replicar um layout idêntico ao exemplo. Utilização das tags html, head e body; utilização de classes no CSS para aplicar os estilos. Referência ao arquivo CSS com a tag link.

Repositório Github Page

Projeto Mundo dos Vinhos

Aprofundamos no conteúdo de layout de página com o aprendizado de tags semânticas, que descrevem significado de conteúdo e não apenas indicam aparência visual. Recebemos um modelo de estrutura de layout, a orientação sobre utilizar sites que disponibilizam imagens gratuitas e o desafio de elaborar uma página com tema livre utilizando o conteúdo aprendido.

Repositório Github Page

Projeto Blue

Neste momento começamos a utilizar flexbox para organizar conteúdos na tela de modo geral. Imagens lado a lado, ou em coluna. Também tivemos uma área para logo e menu. Foi solicitado que, ao passar o mouse em cima das imagens, elas tivessem uma transição de cor para preto e branco.

Repositório Github Page

Projeto Mercados Públicos - Recife

A partir deste desafio, pusemos em prática a responsividade. O objetivo era de que adaptássemos o conteúdo da nossa tela em desktop para ser visualizado em telas mobile. Acrescentei um áudio, pois vi que havia identificação com o tema que escolhi. Vimos também meta tag viewport e media query (usada em css para aplicar regras específicas de estilo de acordo com as características de um dispositivo).

Repositório Github Page

Projeto O Senhor dos Anéis

Foi um grande projeto! Uau… Começamos a fazer um site responsivo com o conceito do mobile first no CSS. De fato, com o uso dos smartphones é muito comum o acesso a sites através do aparelho. É mais fácil e está quase todo o tempo em mãos. No entanto, ao aprender sobre essa estratégia, pude perceber que muitas empresas ainda não a valorizam, o que prejudica bastante a visualização das suas respectivas páginas pelo celular. Também utilizamos o arquivo normalize.css, com o objetivo de padronizar os elementos HTML em diferentes navegadores.

Repositório Github Page

Projeto JavaScript

Usando um pouquinho de HTML e CSS, o foco desse projeto foi iniciar JavaScript. Aprendemos algumas noções teóricas, utilidade do JS, conceito de variáveis e seus tipos, operadores aritméticos, operadores relacionais, estruturas condicionais, operadores lógicos. O nosso desafio foi escrever um programa que solicitasse ao usuário um número inteiro e verificasse se o número atendia aos critérios pré-determinados.

Repositório Github Page

Projeto Calculadora DOM

A aula 12 do curso nos apresentou o DOM (Document Object Model), um recurso bastante útil, pois, de forma resumida, permite que o JavaScript acesse, modifique e manipule elementos HTML. Com o DOM é possível criar web sites interativos, trazendo mais dinamismo. Aprendemos como ver a árvore do DOM de qualquer site, selecionar elementos, tais como ID, Classe e Seletor, também conhecemos os Eventos. Nosso desafio foi criar uma calculadora, selecionando uma div, adicionando 2 eventos (listeners) e chamando funções.

Repositório Github Page

Projeto Sonic

Para esse desafio, o objetivo era criar um menu hamburguer seguindo o conceito de mobile first e utilizando variáveis, evento e função em JavaScript.

Repositório Github Page

Projeto Bootstrap Carrossel

E chegamos ao último desafio do nosso curso: utilizar o framework Bootstrap. Aprendemos sobre a sua instalação/utilização, os arquivos css e js, a verificação dos arquivos, os containers... A proposta para o último projeto foi a criação de um carrosel utilizando o Bootstrap. Na primeira página, criaríamos um botão que, ao clicar, direcionaria para uma nova página, galeria, com um carrossel de imagens.
